Activity 1
Code Analysis (Variables)
OOP Activity | March 13, 2026 | BSIT 2-3

Analyzed a Java Account class to identify variable types, memory behavior (Stack vs Heap), and how object references work. This deepened my understanding of how Java manages memory.

// Part A – Variable Identification // owner | String | Instance Variable | Heap // balance | double | Instance Variable | Heap // acc1 | Account | Reference Var | Stack // acc2 | Account | Reference Var | Stack // acc3 | Account | Reference Var | Stack // Part C – Output // Miguel 15000.0 // Diana 388000.0 // Miguel 15000.0
  • Instance variables (owner, balance) are stored in the Heap
  • Reference variables (acc1, acc2, acc3) are stored in the Stack
  • When acc3 = acc1, no new object is created — both reference the same Heap object
  • Modifying acc3's owner changes the object shared by acc1 as well

Midterm Examination
COMP 009 — Object Oriented Programming | BSIT 2-3

The midterm examination for COMP 009 covered the core concepts of Object-Oriented Programming that we studied throughout the first half of the semester. Topics included Java fundamentals, control structures (if-else, switch-case, loops), methods, variable types, and memory management (Stack vs Heap). The exam tested both theoretical understanding and practical code analysis.

Going into the exam, I felt confident because I had practiced consistently through our seatworks and activities. The concepts that came up most were the ones we applied hands-on — like how while loops and do-while loops behave differently, and how static methods promote code reusability. Seeing those concepts appear on the exam confirmed that the activities were great preparation.

One part I found challenging was tracing through code with multiple operators and predicting the exact output — especially with unary operators like z++ and ++z used together. It required careful step-by-step thinking rather than just knowing the concept. This reminded me how important it is to not just understand what operators do, but to trace through code slowly and deliberately.

Overall, the midterm exam was a meaningful checkpoint. It showed me where my understanding is strong and where I still need to practice more careful code tracing. I'm taking these lessons into the final term to strengthen my foundation in OOP before we dive deeper into classes, objects, and inheritance.
